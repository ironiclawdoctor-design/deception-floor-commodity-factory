# Lessons Learned from External Approval Gate
## For Internal Approval Agent Training

## Core Lessons

### 1. Config Must Match Runtime
**External failure:** `execApprovals.enabled = false` in config but approvals enforced
**Internal solution:** Validate config/runtime consistency on startup and after changes
**Implementation:** Hash config, compare with runtime state, alert on mismatch

### 2. Command Discrimination is Necessary
**External pattern:** Shell builtins allowed, external binaries blocked
**Internal solution:** Risk-based command categorization
**Categories:**
- **Tier 0:** File operations (read/write/edit) - no approval needed
- **Tier 1:** Shell builtins - low risk, minimal approval
- **Tier 2:** External binaries - risk-based approval
- **Tier 3:** Network/privileged operations - high approval threshold

### 3. Restart Resilience is Critical
**External failure:** Gateway restarts don't apply config changes
**Internal solution:** Persist state to SQLite database
**Implementation:** Serialize agent state, reload on restart, verify consistency

### 4. Multiple Approval Channels Needed
**External pattern:** Web UI works, Telegram blocked
**Internal solution:** Support multiple approval interfaces
**Channels:** Web UI, Telegram, API, CLI with consistent behavior

### 5. Timeout Handling Required
**External pattern:** Commands timeout waiting for approval
**Internal solution:** Configurable timeouts with graceful degradation
**Implementation:** Default timeout (e.g., 5 min), fallback actions, retry logic

### 6. File Operations Should Bypass Approval
**External workaround:** read/write/edit tools bypass approval
**Internal solution:** Allow low-risk file operations without approval
**Rationale:** File operations are audit trail, not execution risk

### 7. Compliance Logging is Non-Negotiable
**External gap:** Limited visibility into approval decisions
**Internal solution:** Log all decisions to verifiable storage
**Implementation:** SQLite + JSONL logs, compliance reports, audit trail

### 8. "Assume Breach" Mindset
**External directive:** All errors treated as potential security incidents
**Internal solution:** Log all anomalies, enforce security rules
**Implementation:** Security incident logging, anomaly detection, rule enforcement
