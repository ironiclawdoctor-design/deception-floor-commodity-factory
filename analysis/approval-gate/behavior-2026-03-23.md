## Executive Summary
The external approval gate demonstrates consistent misbehavior: config says `execApprovals.enabled = false` but runtime enforces approvals. This analysis extracts patterns to train our internal approval agent.

## Core Anomaly: Config/Runtime Mismatch
- **Config state:** `execApprovals.enabled = false` at both top-level and accounts.default
- **Runtime behavior:** Still enforces approvals for non-builtin commands
- **Error message:** "Exec approval is required, but chat exec approvals are not enabled on Telegram"
- **Persistence:** Survives gateway restarts (PID changes, config updates)

## Behavioral Patterns

### Pattern 1: Command Discrimination
**Blocked commands:** `ls`, `find`, `openclaw`, `tail`, `jq`, `grep`, `sqlite3`, `chmod`, `ln`, `mv`, `cp`
**Allowed commands:** `date`, `echo`, `pwd`, `cd` (shell builtins)
**Discrimination basis:** External binaries vs shell builtins
**Workaround:** File operations via `read`/`write`/`edit` tools bypass approval

### Pattern 2: Gateway Restart Ineffectiveness
**Observed:** Gateway PID changes (2092799→2093816→2095603) but config not respected
**Config updates:** Applied at 15:36:37 and 16:05:36 UTC
**Result:** Approval gate remains active despite restarts
**Hypothesis:** Container environment, service vs foreground mode issues

### Pattern 3: Async Timeout Pattern
**Examples:**
1. `openclaw doctor` - job eb1a45e3 (15:42 UTC)

**Pattern:** Commands timeout waiting for approval that never arrives
**Implication:** Approval IDs may expire before human can approve

### Pattern 4: Web UI Terminal Exception
**Observation:** Web UI terminal provides interactive approval while Telegram blocks
**Evidence:** Human directed to use Web UI for commands needing approval
**Implication:** Approval mechanism varies by client interface

### Pattern 5: File Operation Bypass
**Observation:** `read`, `write`, `edit` tools bypass approval gate
**Evidence:** Script creation, config updates, log reading all work without approval
**Implication:** Gateway approval only applies to shell exec, not file operations
