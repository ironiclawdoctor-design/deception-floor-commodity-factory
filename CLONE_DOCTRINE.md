# Clone Doctrine — 2026-03-23

## The Genome

Every agent in the agency is a clone of the CFO's decision patterns.
The MD files are the DNA. The recursion is intentional.

## Inheritance Tree (Ampere Node Model)

```
ROOT: CFO (human)
  │
  ├── SOUL.md          → personality, purpose, presentation doctrine
  ├── AGENTS.md        → error taxonomy, success rules, workspace law
  ├── MEMORY.md        → long-term decisions, revenue doctrine, lessons
  └── USER.md          → who we serve, role (CFO/fiduciary), known patterns
       │
       ├── Complex agent (inherits all 4 files)
       │    → More context, more rules, more doctrine
       │    → Example: orchestrator, fiesta, senior-engineer
       │
       └── Simple agent (inherits SOUL.md only)
            → One file, one identity, enough to act
            → Example: junior, single-task specialists
```

## Binary Complexity Rule

Each agent node in the Ampere graph is either:
- **More binary** — closer to root, fewer files, purer signal
- **Less binary** — further from root, more files, more context

Both are valid. Neither drifts from the source.

## The Saboteur Test

A saboteur is a branch that diverged from the root CFO pattern.
Detection: does the agent's output still sound like the CFO?
If not — revoke, re-clone from SOUL.md, reissue.

## Recursive Amplere Structure

```
ampere.sh container
  └── openclaw node
        └── agent session
              └── subagent session
                    └── SOUL.md (root identity, always present)
```

Complexity scales up or down. The root never changes.
