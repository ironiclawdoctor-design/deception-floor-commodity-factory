### Step 2B: Accounting Infrastructure Built (Late afternoon)
**Problem:** Wallet without double-entry accounting is a black hole (deposits disappear, no audit trail).

**What was built:**
- `accounting.sql` — Double-entry ledger schema (SQLite)
  - Accounts (chart of accounts: assets, liabilities, revenue, expenses, equity)
  - Journal entries (balanced transactions)
  - Deposits & settlements (source data)
  - Reconciliations (monthly blockchain vs. ledger verification)
  - Variances (flags discrepancies automatically)
  - Views: trial balance, deposit summary, settlement summary, variance report

- `ledger.sh` — Ledger entry tool
  - `ledger.sh deposit <amount> <type> <tx_hash> <from> <to>` — Record crypto deposits (creates balanced DR/CR entry)
  - `ledger.sh settlement <amount> <tokens> <invoice>` — Record Ampere settlements (balanced DR/CR)
  - `ledger.sh balance` — Show trial balance (debits - credits must = 0)
  - `ledger.sh deposits` — List all deposits
  - `ledger.sh settlements` — List all settlements

- `reconcile.sh` — Reconciliation tool
  - Queries blockchain for actual wallet balance (via free APIs: Etherscan, Blockchair, Solana.fm)
  - Queries SQLite for ledger balance
  - Flags variances automatically (triggers alerts if >$0.01 difference)
  - Logs reconciliation to reconciliation.jsonl (auditable)
  - Inserts variance records to database for investigation

**Cost:** $0.00 (SQLite, bash, free blockchain APIs)

**Audit ready:** Every deposit and settlement is balanced. Monthly reconciliation proves integrity. Variances are flagged immediately.

**Tax compliance:** IRS-ready records (double-entry, categorized by revenue type, donation tracking)
