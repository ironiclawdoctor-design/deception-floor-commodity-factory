## 6 Questions Answered

**1. Surname rejection risk — Section 2(e)(4)?**
HIGH RISK. "Shan" is a recognizable surname (Chinese, Irish "O'Shaughnessy" shortened, South Asian). USPTO applies a 5-factor test (frequency in phone directories, appearance as surname in public, whether anyone in the applicant's company has the name, whether it has any other meaning, etc.). "Shan" has ~50K+ bearers in US surname databases. Likely triggers a §2(e)(4) Office Action. Rebuttal requires evidence that "Shan" has acquired distinctiveness (secondary meaning) or an alternative meaning that dominates public perception (e.g., a unit of information entropy = Shannon/Shan).

**2. Relevant USPTO Classes for a digital currency/ledger unit?**
- **Class 36** — Financial services; digital currency; cryptocurrency exchange; virtual currency
- **Class 42** — Software as a service; computer programming; ledger technology; blockchain services
- **Class 9** — Downloadable software; digital tokens; electronic ledger software
*Recommend filing in Class 36 + 42 minimum. Class 9 if distributing software.*

**3. Existing "Shan" marks likely to cause confusion?**
- **SHAN** (apparel/fashion) — likely live marks in Class 25 (less relevant)
- **SHAN** (food/restaurant) — Class 43 (less relevant)
- **Financial/software space:** Search tmsearch.uspto.gov for "SHAN" in IC 036 and IC 042 before filing. Any live mark in fintech/crypto = §2(d) likelihood-of-confusion block. Risk is moderate-high given crypto mark proliferation since 2018.

**4. TEAS Plus cost and timeline?**
- **TEAS Plus fee:** $250/class (as of 2024 fee schedule; verify at uspto.gov/trademarks/apply/fees)
- 2 classes (036 + 042) = **$500 filing fee**
- **Timeline:** ~8–12 months to registration if no Office Actions; 12–18 months with one OA response
- TEAS Plus requires: pre-approved ID Manual description, email correspondence, no amendments to identification

**5. Use-in-commerce vs intent-to-use (ITU)?**
For an **internal ledger unit** not yet publicly marketed:
- **ITU (§1(b))** is correct — the Shan unit exists internally but has not been used in interstate commerce in connection with financial services offered to the public
- Use-in-commerce (§1(a)) requires actual commerce: offering the service to third parties, not just internal use
- ITU buys 6 months (extendable to 36 months) to achieve commercial use before Statement of Use is required
- **Caution:** An internal-only unit may face a bona fide intent challenge if USPTO questions whether commercial use is genuinely planned

**6. Fastest registration path?**
1. Conduct TESS search first (tmsearch.uspto.gov) — clear Class 36/42 conflicts
2. File TEAS Plus ITU in Class 36 + 42 ($500)
3. Prepare a "non-surname" argument: "Shan = unit of information entropy (Claude Shannon) = technical term with primary non-surname meaning" — this directly rebuts §2(e)(4)
4. File with specimen plan in place for Statement of Use
5. If budget allows: add Class 9 for software ($250 more)
*Fastest = ~8 months if no OA. Surname argument prepared upfront = reduces OA round-trip.*
