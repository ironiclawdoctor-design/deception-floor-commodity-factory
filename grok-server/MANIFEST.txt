================================================================================
API ENDPOINTS
================================================================================

GET /health
  Returns: {"status": "healthy", "model": "grok-bash-1.0", "cost": "$0.00"}

GET /status  
  Returns: {"requests": N, "inferences": M, "port": 8888}

POST /infer
  Request: {"prompt": "your question"}
  Returns: {"response": "...", "tokens_used": 0, "cost": "$0.00"}
  Pattern Examples:
    - "bash" → "bash is the firewall..."
    - "token" → "zero tokens. infinite bash. victory."
    - "time" → Current time + sarcasm
    - "weather" → Random temp + sarcasm
    - Default → Generic "thinking..." response

GET /metrics
  Returns: Prometheus-format metrics (requests, cost, sovereignty)

GET /
  Returns: HTML documentation

================================================================================
ARCHITECTURE
================================================================================

Request Flow:
  HTTP Client
      ↓
  Python http.server (port 8888)
      ↓
  GrokHandler.do_GET/do_POST()
      ↓
  grok_infer() [bash logic in Python]
      ↓
  Pattern matching on prompt
      ↓
  JSON response
      ↓
  HTTP Client

No external APIs. No token consumption. No cloud dependencies.
