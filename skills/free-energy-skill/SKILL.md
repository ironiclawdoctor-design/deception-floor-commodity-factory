# free-energy-skill

```yaml
name: free-energy-skill
description: Meta-skill that audits, improves, and generates new AgentSkills. Use when: (1) auditing all installed skills for quality gaps, (2) improving a specific skill's SKILL.md beyond its current capability, (3) generating a new skill from a domain description, (4) the agency needs more capability than current skills provide. Named after thermodynamic free energy — outputs more useful work than activation cost. Triggers on: "improve all skills", "audit skills", "generate skill", "skill quality", "free energy", "skill gap".
```

## What This Is

A meta-skill that generates more agency capability than it costs to activate. Every run produces at least one improved SKILL.md, one gap diagnosis, or one new skill draft. The cost: one activation. The output: N improvements.

Named after thermodynamic free energy: ΔG = ΔH - TΔS. The useful work extracted from the system exceeds the energy put in, because the system has internal structure that amplifies the input.

The internal structure here: pattern libraries, caches, and a scoring rubric that converts vague quality intuitions into executable diffs.

---

## Modes

### Mode 1: Full Audit
> "audit all skills" / "skill quality check" / "free energy run"

Scans every skill in the workspace. Scores each on 5 dimensions. Outputs a ranked list lowest-first (most improvable first). Saves to `audit-cache.json`.

```bash
python3 scripts/audit_skills.py
```

Output: ranked table + audit-cache.json

### Mode 2: Improve a Specific Skill
> "improve skill [name]" / "fix [skill] SKILL.md"

Reads an existing SKILL.md. Diagnoses gaps. Outputs an improved version to stdout without overwriting.

```bash
python3 scripts/improve_skill.py --path /path/to/skill/
# Preview only. Add --apply to overwrite.
python3 scripts/improve_skill.py --path /path/to/skill/ --apply
```

### Mode 3: Generate a New Skill
> "generate skill for [domain]" / "create skill that does X"

Describe the domain in natural language. The agent reads `references/skill-patterns.md` and produces a spec-compliant SKILL.md scaffold. Then fills in the scaffold with domain-specific content.

Steps:
1. Read `references/skill-patterns.md`
2. Draft SKILL.md using the 5 patterns
3. Write to `skills/[name]/SKILL.md`
4. Add to audit-cache.json with initial score

### Mode 4: Gap Detection
> "what skills are missing" / "skill gap"

Reads the workspace's AGENTS.md, SOUL.md, and memory files. Cross-references against installed skills. Identifies capability gaps. Proposes new skill names and descriptions.

## Scoring Rubric

Each skill is scored 0-100 across 5 dimensions (20 pts each):

| Dimension | 0 pts | 10 pts | 20 pts |
|---|---|---|---|
| **Description Clarity** | Vague, no triggers | Partial triggers | Exact trigger phrases, never-use cases |
| **Workflow Specificity** | "use the tool" | Named steps | Numbered steps with exact commands |
| **Bundled Resources** | Nothing | One script | scripts/ + references/ populated |
| **Progressive Disclosure** | Wall of text | Some structure | Quick-start → full spec → edge cases |
| **Token Efficiency** | >600 lines | 300-600 lines | <300 lines without losing function |

Skills scoring < 50 are **below free energy threshold** — they cost more tokens to parse than they return in capability.

## Workflow

```
1. Read this SKILL.md
2. Choose mode based on trigger phrase
3. Run the appropriate script or follow steps
4. For audit: report 5 lowest-scoring skills
5. For improve: show diff, ask for --apply confirmation
6. For generate: write new skill, add to cache
7. Update audit-cache.json with new scores
```

## Edge Cases

- **Skill with no SKILL.md**: scored 0, flagged as `missing`. Generate one using Mode 3.
- **Skill with only frontmatter**: scored on frontmatter quality only. Likely < 20.
- **Circular improvement**: if improve_skill.py is run on itself, it reads its own patterns and outputs the same content. This is correct — it means the skill is already at threshold.
- **Cache staleness**: audit-cache.json has a `generated_at` timestamp. If >7 days old, re-run audit before using scores.

---

## Free Energy Theorem

A skill exceeds free energy threshold if and only if:
> Score > 60 AND has at least one item in scripts/ OR references/

Below this line, the skill costs more to parse than it saves. Above it, every activation returns more capability than it consumes.

This skill's own score target: **85/100**.
