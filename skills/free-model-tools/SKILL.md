# Free-Model Tools — Workaround Architecture

## The Problem

Agency crons run on `openrouter/z-ai/glm-4.5-air:free` and other free tier models. These models have no tool endpoint. When `tools` parameter is sent:

```
404: No endpoints found that support tool use
```

This is SR-022. The card is named **Sideload** in the agency deck. The model can think. It cannot call tools. You must supply the tools differently.

---

## Phase 0: What "No Tools" Actually Means

| Capability | Free Model | Workaround |
|------------|------------|------------|
| Read files | ❌ native | ✅ via bash in prompt |
| Write files | ❌ native | ✅ via bash in prompt |
| SQLite query | ❌ native | ✅ via `sqlite3` in prompt |
| API calls | ❌ native | ✅ via `python3 -c` or `curl` in prompt |
| OpenClaw tools | ❌ (session has tools, model can't invoke) | ✅ structured output → parent agent executes |
| Tool endpoint (404) | ❌ hard 404 | ✅ never send `tools` param to free models |

**What actually breaks when `tools` parameter is sent:**
- OpenRouter returns HTTP 404 (not 400, not 422 — 404)
- The cron registers a `consecutiveErrors` increment
- No output is produced — the call fails before model runs
- SR-022: Use `glm-4.5-air:free` which has `agentTurn` support WITHOUT requiring tool endpoint

---

## Pattern 1: Bash-as-Tool

The entire prompt payload IS the tool call. The model receives: "run this bash, return the output." No tool endpoint needed.

### Complete Vocabulary

#### File Read

```bash
# Safe (tail only — full reads timeout on glm:free at 400s budget)
tail -20 /root/.openclaw/workspace/memory/2026-03-27.md

# Safe (grep for signal)
grep "CFO\|famine\|urgent" /root/.openclaw/workspace/memory/2026-03-27.md | tail -5

# Safe (wc for progress checks)
wc -l /root/.openclaw/workspace/some-progress-file.md

# UNSAFE — full read times out
cat /root/.openclaw/workspace/AGENTS.md   # 50KB+ will timeout
```

**Limitation:** Full file reads fail on large files within 400s budget. Always `tail` or `grep`. Skill file reads don't survive at all (`wifehusband` failure, 2026-03-26).
**Escape hatch:** If full content needed, parent agent (Claude) reads it and summarizes into cron prompt.

---

#### File Write

```bash
# Simple overwrite
echo "STATUS: clean" > /root/.openclaw/workspace/status.txt

# Append
echo "$(date -u +%Y-%m-%dT%H:%M:%SZ) — event logged" >> /root/.openclaw/workspace/log.txt

# Multi-line via heredoc
cat > /root/.openclaw/workspace/output.md << 'EOF'
# Report
Generated by cron agent.
EOF

# Write JSON
python3 -c "
import json, datetime
data = {'ts': datetime.datetime.utcnow().isoformat()+'Z', 'status': 'ok'}
with open('/root/.openclaw/workspace/cron-state.json', 'w') as f:
    json.dump(data, f)
"
```

**Limitation:** Paths must be within `/root/.openclaw/workspace/` or `/root/human/` (PL-002). No glob patterns in Telegram exec (SR-011).
**Escape hatch:** Write to temp path, rename atomically with `mv`.

---

#### SQLite Query

```bash
# Quick read (SR-001: bypasses approval gate)
sqlite3 /root/.openclaw/workspace/agency.db "SELECT value FROM ledger WHERE key='balance' LIMIT 1;"

# Conditional read
sqlite3 /root/.openclaw/workspace/agency.db "SELECT COUNT(*) FROM confessions WHERE ts > datetime('now', '-24 hours');"

# Write (Python for safety — SR-001)
python3 -c "
import sqlite3
conn = sqlite3.connect('/root/.openclaw/workspace/agency.db')
conn.execute('INSERT INTO log (ts, event) VALUES (datetime(\"now\"), \"cron ran\")')
conn.commit()
conn.close()
"
```

**Limitation:** Complex joins + file I/O in same cron → timeout. One sqlite call per cron. Long/write ops use Python.
**Escape hatch:** Pre-materialize complex queries into a cache file; cron reads cache.

---

#### API Call

```bash
# Python urllib (no pip needed — stdlib)
python3 -c "
import urllib.request, json
req = urllib.request.Request('https://api.example.com/status',
    headers={'Authorization': 'Bearer TOKEN'})
resp = urllib.request.urlopen(req, timeout=10)
data = json.loads(resp.read())
print(data.get('status', 'unknown'))
"

# curl (if available)
curl -s --max-time 10 "https://api.example.com/ping" | python3 -c "import json,sys; print(json.load(sys.stdin).get('ok'))"

# BTC price fetch (AR-006 pattern — avoid in crons; use cache)
python3 -c "
import urllib.request, json
r = urllib.request.urlopen('https://api.coindesk.com/v1/bpi/currentprice/USD.json', timeout=8)
d = json.loads(r.read())
print(d['bpi']['USD']['rate_float'])
" > /root/.openclaw/workspace/btc-price-cache.txt
```

**Limitation:** HTTP calls inside 90s budget are high-risk (AR-006). Network latency on ChAmpEredar container + free model overhead = frequent timeout.
**Escape hatch:** Separate cron writes cache; signal cron reads file. Never make HTTP call inside signal cron.

---

#### Conditional Logic

```bash
# Simple if/else
VALUE=$(sqlite3 /root/.openclaw/workspace/agency.db "SELECT balance FROM ledger LIMIT 1;" 2>/dev/null)
if [ -z "$VALUE" ] || [ "$VALUE" -lt "100" ]; then
    echo "ALERT: balance below threshold"
else
    echo "CLEAN"
fi

# File existence check
if [ -f "/root/.openclaw/workspace/secrets/api-key.txt" ]; then
    echo "KEY: present"
else
    echo "KEY: missing — cron should not proceed"
fi
```

**Limitation:** Complex multi-branch logic = model interprets/executes inconsistently. Keep to 2 branches max per cron. If >2 branches needed: use Python.

---

#### JSON Parsing

```bash
# Parse from file
python3 -c "
import json
with open('/root/.openclaw/workspace/domain-registry.json') as f:
    d = json.load(f)
print(d.get('dashboard', {}).get('current_domain', 'unknown'))
"

# Parse from stdin (pipe from API call)
echo '{"status":"ok","balance":42}' | python3 -c "import json,sys; d=json.load(sys.stdin); print(d['balance'])"

# Append to JSONL
python3 -c "
import json, datetime
entry = {'ts': '$(date -u +%Y-%m-%dT%H:%M:%SZ)', 'event': 'cron-ran', 'status': 'clean'}
with open('/root/.openclaw/workspace/cron-log.jsonl', 'a') as f:
    f.write(json.dumps(entry) + '\n')
"
```

**Limitation:** Large JSON files → parse fails silently or model truncates. Always `.get()` with defaults; never assume keys exist.

### Bash-as-Tool Prompt Template

```
[AGENT_NAME] check. Run: [SINGLE BASH COMMAND OR CHAIN].
Report result in one sentence: "[LABEL]: [value]". If command fails, write "ERROR: [reason]".
```

**Non-negotiables:**
- One action per cron (AR-005 doctrine)
- No skill file reads in payload (Gideon Test rule 3)
- Output format explicit ("one sentence: LABEL: value")
- Timeout budget: bash_exec_time + model_overhead (20-40s) + buffer = total
  - Pure bash: 30s ceiling (FX-009)
  - Bash + sqlite: 45s ceiling (FX-010)
  - Bash + python: 60s ceiling
  - Bash + HTTP call: NEVER — use cache pattern

## Pattern 2: Structured Output → Parent Agent Executes

When the task is too complex for bash-as-tool, or when the free model needs to make a decision that requires actual tool execution: use structured output protocol.

### Protocol

1. **Free model** receives prompt: describe what you need done, output as JSON
2. **Free model** outputs action JSON (cannot execute it)
3. **Parent agent** (Claude, with tools) reads JSON, executes on model's behalf
4. Parent feeds result back if needed

### Complete Action Schema

```json
{
  "action": "file_write",
  "path": "/root/.openclaw/workspace/output.md",
  "content": "# Report\nGenerated content here.",
  "mode": "overwrite"
}
```

```json
{
  "action": "file_read",
  "path": "/root/.openclaw/workspace/memory/2026-03-27.md",
  "tail": 20
}
```

```json
{
  "action": "exec_command",
  "command": "sqlite3 /root/.openclaw/workspace/agency.db \"SELECT COUNT(*) FROM ledger\"",
  "timeout_seconds": 10
}
```

```json
{
  "action": "api_call",
  "method": "GET",
  "url": "https://api.coindesk.com/v1/bpi/currentprice/USD.json",
  "headers": {},
  "timeout_seconds": 8,
  "extract_path": "bpi.USD.rate_float"
}
```

```json
{
  "action": "cron_update",
  "job_id": "ee30d32e-7e3e-437c-bd01-92a346b3cac6",
  "patch": {
    "timeoutSeconds": 30,
    "enabled": true
  }
}
```

```json
{
  "action": "memory_append",
  "file": "/root/.openclaw/workspace/memory/2026-03-27.md",
  "content": "\n## Cron Check\n- Status: clean\n- Timestamp: 2026-03-27T14:00:00Z"
}
```

### Full Schema Reference

| Action | Required Fields | Optional Fields |
|--------|----------------|-----------------|
| `file_write` | `path`, `content` | `mode` (overwrite/append) |
| `file_read` | `path` | `tail` (lines), `grep` (pattern) |
| `exec_command` | `command` | `timeout_seconds`, `cwd` |
| `api_call` | `method`, `url` | `headers`, `body`, `timeout_seconds`, `extract_path` |
| `cron_update` | `job_id`, `patch` | none |
| `memory_append` | `file`, `content` | none |
| `db_write` | `db_path`, `sql` | none |
| `send_alert` | `message` | `channel` (telegram default) |

### Parent Agent Handler Pattern

When you receive structured output from a free-model cron:

```python
# Pseudocode — parent agent (Claude) executes this logic

def handle_free_model_output(raw_output: str):
    # Extract JSON from model output (may have prose around it)
    json_match = re.search(r'\{.*\}', raw_output, re.DOTALL)
    if not json_match:
        log("No structured output found — treating as informational")
        return
    
    action = json.loads(json_match.group())
    
    match action["action"]:
        case "file_write":
            write(path=action["path"], content=action["content"])
        case "file_read":
            return read(path=action["path"], tail=action.get("tail"))
        case "exec_command":
            return exec(command=action["command"])
        case "api_call":
            return web_fetch(url=action["url"])
        case "cron_update":
            return cron(action="update", jobId=action["job_id"], patch=action["patch"])
        case "memory_append":
            # Use edit tool to append
            return edit(file_path=action["file"], newText=action["content"])
        case "send_alert":
            return message(action="send", target="telegram", message=action["message"])
```

**Parser hardening:** Always `re.search` for JSON — free models add prose around their output. Strip code fences. Use `.get()` with defaults on all fields.

## FT-Series Rules (Free-Model Tools)

### FT-000: Never Send `tools` Parameter to Free Models
**Limitation:** Free models return HTTP 404 when `tools` parameter included in request.
**Workaround:** Remove `tools` from request. Use bash-as-tool or structured output patterns.
**Failure mode of workaround:** Model may hallucinate tool calls in text form. Detect via regex; treat as structured output.

### FT-001: Bash-as-Tool is the Primary Pattern
**Limitation:** No native tool endpoint on free models.
**Workaround:** Entire prompt = bash script. Model executes, returns output.
**Failure mode:** Model adds prose around bash output. Always parse `LABEL: value` format. Enforce with explicit output format in prompt.

### FT-002: Tail, Don't Cat
**Limitation:** Full file reads + model inference exceeds 400s budget. `cat AGENTS.md` on glm:free = timeout.
**Workaround:** `tail -20 <file>` or `grep <pattern> <file> | tail -5`.
**Failure mode:** Tail misses context at file start. Escape hatch: parent agent (Claude) reads full file, summarizes key facts into cron prompt at deploy time.

### FT-003: One Command Per Cron
**Limitation:** Sequential bash commands accumulate latency. Each `&&` adds execution time.
**Workaround:** One action per cron. Split multi-step tasks into multiple crons.
**Failure mode:** Cron proliferation. Mitigate with CR-013 (redundancy detection) before adding new crons.

### FT-004: No HTTP Inside Signal Crons
**Limitation:** HTTP calls inside 90s budget = frequent timeout (AR-006).
**Workaround:** Separate cache-writer cron (runs every 5-15min, writes to file). Signal cron reads file.
**Failure mode:** Cache goes stale if cache-writer also times out. Add `mtime` check: if cache > 30min old, output "STALE: cache expired" and alert parent.

### FT-005: Model Overhead Budget
**Limitation:** `glm-4.5-air:free` adds 20-40s overhead per call regardless of bash complexity.
**Workaround:** Budget = bash_exec_time + 35s (median overhead) + 10s buffer.
  - Pure bash: 30s ceiling (FX-009 confirmed)
  - Bash + sqlite: 45s ceiling (FX-010 confirmed)
  - Bash + python: 60s ceiling
**Failure mode:** Free tier congestion → overhead spikes to 60s+. Detect via `consecutiveErrors ≥ 2` → increase timeout by 50% once. If still failing: model is overloaded, not the bash.

### FT-006: Structured Output Requires Extraction, Not Assumption
**Limitation:** Free models add prose, disclaimers, and markdown around JSON output.
**Workaround:** Use `re.search(r'\{.*?\}', output, re.DOTALL)` to extract. Never `json.loads(raw_output)` directly.
**Failure mode:** Nested JSON with prose mid-object. Use more aggressive: `re.search(r'\{[^}]*\}', output)` for simple objects; full JSON decoder with error recovery for nested.

### FT-007: Credentials Are Not in Free Model Scope
**Limitation:** Free models cannot access secrets vault, env vars, or encrypted files natively.
**Workaround:** Parent agent (with tools) pre-reads credential and injects value into cron prompt at deploy time. Or: store credential in plaintext file with path in prompt (PL-002 scoped to workspace only).
**Failure mode:** Credential rotation invalidates injected value. Use credential file path, not value: `$(cat /root/.openclaw/workspace/secrets/key.txt)` in bash.

### FT-008: Structured Output Is the Bridge, Not the Destination
**Limitation:** Structured output → parent execution adds 1-2 RTTs versus native tool call.
**Workaround:** Use structured output only for: decisions that require tool execution, multi-step workflows, anything needing OpenClaw tools (cron_update, message, etc.).
**Failure mode:** Parent agent timeout before executing all actions. Always process actions sequentially; log partial completion; resume from last successful action.

### FT-009: Free Tier Rate Limits Are Silent Killers
**Limitation:** `qwen3-coder:free` and similar have RPM caps not documented in error messages. Hit cap → silent empty output.
**Workaround:** Stick to `glm-4.5-air:free` for all isolated crons. It has the highest observed free-tier RPM.
**Failure mode:** Even glm:free has limits. If empty output on two consecutive runs: check OpenRouter status, not the bash.

### FT-010: Skill Files Don't Survive in Cron Payload
**Limitation:** Including skill file path/content in cron prompt → timeout. Skill reads add 50-100s to budget.
**Workaround:** Distill skill to 3-5 line brief. Move brief directly into payload. Skills are for main session only (Gideon Test rule 3).
**Failure mode:** Distilled brief misses edge cases. Mitigation: parent agent reviews first run output; updates brief if edge case hit.

### FT-011: The 404 Is Not a Retry Signal
**Limitation:** When `tools` param sent to free model → 404 → OpenClaw logs `consecutiveErrors++`.
**Workaround:** Immediately remove `tools` param from cron config on first 404. Do not retry with same config.
**Failure mode:** Config not updated before next cron fire. Use cron `update` to patch model AND remove tools config atomically.

### FT-012: Python3 stdlib Only — No pip
**Limitation:** Free model crons run in isolated sessions. No pip, no venv, no third-party packages.
**Workaround:** Use only Python3 stdlib: `urllib.request`, `json`, `sqlite3`, `datetime`, `re`, `os`, `sys`, `subprocess`.
**Failure mode:** Model generates `import requests` (muscle memory from training data). Catch via prompt: "Use only Python3 stdlib. No pip. No requests library."

## Deployment Checklist for Free-Model Crons

Before deploying any cron on a free model, verify:

```
□ Model: openrouter/z-ai/glm-4.5-air:free
□ No `tools` parameter in cron config
□ sessionTarget: "isolated"
□ Payload is ≤ 5 lines (brief, not skill file)
□ Output format explicit: "one sentence: LABEL: value"
□ Timeout budget: bash_exec_time + 35s overhead + 10s buffer
□ No HTTP calls (use cache file if needed)
□ No full file reads (tail or grep only)
□ One action per cron (AR-005)
□ Reactivation trigger documented (Gideon Test)
□ Silent on success (no announce on clean run)
```

```
You are [AGENT_NAME]. Your task: [MANDATE].

After completing your analysis, output a single JSON object (no prose, no code fences):
{"action": "file_write", "path": "/root/.openclaw/workspace/[OUTPUT_PATH]", "content": "[YOUR_OUTPUT]"}

If no action needed, output: {"action": "none", "reason": "[WHY_CLEAN]"}
```
