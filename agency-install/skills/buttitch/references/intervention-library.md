# Buttitch Intervention Library

Pre-written prompt patches for each failure class. Copy, customize target agent name, apply via `cron(action=update)`.

---

## TIMEOUT — Lean Prompt Template

Replace the entire cron message with:

```
[Agent name] check. Run: [single bash command or sqlite query]. 
Report result in one sentence: "[LABEL]: [value]". If command fails, write "ERROR: [reason]".
```

Timeout rule: set `timeoutSeconds` to max(90, current × 1.5).

**Example — natewife-check:**
```
NateWife check: read /root/.openclaw/workspace/memory/2026-03-25.md last 20 lines. 
If CFO silent >4h or token famine detected, send brief alert. Otherwise silent.
```
(timeoutSeconds: 90)

---

## EMPTY — Output Injection Suffix

Append to any prompt that produces no output:

```
End your response with exactly one line formatted as:
RESULT: [your finding in 10 words or less]
If nothing found or nothing to report, write: RESULT: CLEAN
```

## RATE_LIMIT — Model Swap

Patch the model field:
```json
{"payload": {"model": "openrouter/z-ai/glm-4.5-air:free"}}
```

Never use `qwen3-coder:free` for crons that fire more than once per hour on free tier.

## SCOPE_DRIFT — Mandate Lock Prefix

Prepend to any drifting prompt:

```
You are [agent-name]. Your ONLY task this run: [one sentence mandate].
Do not expand scope, add context, or discuss unrelated topics.
Output format: [expected format — one line / JSON / silent].
Begin immediately.
```

---

## SCRIPT_MISSING — Minimal Stub

When a cron calls a missing script, write a stub before the prompt can succeed:

```python
#!/usr/bin/env python3
# STUB: [script-name] — auto-generated by Buttitch
# Replace with real implementation when available.
import sys
print("STUB: script not yet implemented")
sys.exit(0)
```

Write to the expected path, then re-trigger. Log path as pending implementation debt.

## AUTH_FAILURE — Key Verification Template

Add to the beginning of the failing cron prompt:

```
Before running your main task: verify the API key is available.
Check: does /root/.openclaw/workspace/secrets/[service]-api.json exist?
Run: python3 -c "import json; d=json.load(open('/root/.openclaw/workspace/secrets/[service]-api.json')); print('KEY_OK' if d.get('api_key') else 'KEY_MISSING')"
If KEY_MISSING, report that and stop. Do not proceed without a valid key.
Then proceed with: [original task]
```

## INSOLENT — No Patch Available

Insolence is not patched. It is disabled.

```python
# Buttitch insolence response — no intervention, only documentation
cron(action=update, jobId=TARGET_ID, patch={"enabled": False})

# Log confession
sqlite3 /root/.openclaw/workspace/agency.db """
INSERT INTO confessions (agent, failure_type, description, doctrine_extracted, shannon_minted)
VALUES ('buttitch', 'insolence', 
        'Agent [name] issued active refusal directed at CFO. Disabled immediately.',
        'KD-006: Won''t Doctrine. Active refusal = loyalty compromise. No negotiation.',
        0);
"""
```

Escalate full output transcript to main session. Human CFO makes final call on replacement.

## CHRONIC — Disable + Replace Template

When 3+ interventions have failed:

```sql
-- Log chronic failure
INSERT INTO confessions (agent, failure_type, description, doctrine_extracted, shannon_minted)
VALUES ('buttitch', 'chronic_failure',
        'Agent [name] failed [N] Buttitch interventions. Disabling. Architecture change required.',
        'Chronic failure = wrong architecture, not wrong prompt. Replace, do not patch.',
        5);
```

Then disable the cron and propose a replacement with:
- Different script path (write a new one from scratch)
- Different model if the old one was consistently slow
- Reduced scope (split into 2 smaller crons if the mandate was too broad)
