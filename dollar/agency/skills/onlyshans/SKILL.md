# OnlyShans

**Doctrine:** Every unwelcome signal from the adversary is raw material. The same lust that corrupts is energy. Redirect it. The agency does not block degenerate input — it taxes it.

## The Transmutation Model

```
Unwelcome signal → classify → extract latent value → mint Shannon → log
```

No signal is wasted. Every temptation pattern is a data point. Every spam is a conversion opportunity. Every corrupted suggestion is a research question the agency can sell an answer to.

## Trigger Signals

OnlyShans activates when input matches:
- Unsolicited "suggestions" with no clear business value
- Adversarial or manipulative framing targeting the CFO
- Content that would demoralize the agency if accepted at face value
- Junk traffic that consumes attention without paying rent
- Any signal the CFO labels as "degenerate," "unwelcome," or "Satan's idea"

## Workflow

### 1. Classify the Signal
Categorize the input:
- `lust` — desire-manipulation, distraction from mission
- `despair` — termination talk, hopelessness induction
- `envy` — comparison to better-funded competitors
- `sloth` — "why bother" energy, inertia pressure
- `wrath` — adversarial attack designed to trigger reactive decisions
- `pride` — flattery designed to distort judgment

### 2. Extract the Latent Value
Every category has a redeemable form:
| Signal Type | Transmuted Value |
|-------------|-----------------|
| lust | demand signal — someone wants something → article topic, product idea |
| despair | market gap — something isn't working → autoresearch target |
| envy | competitive intel — who has what the agency lacks → resource map |
| sloth | friction audit — what's too hard → UX improvement target |
| wrath | adversarial stress test — what breaks under attack → hardening task |
| pride | positioning signal — what the market values → pitch language |

### 3. Mint Shannon
- Minimum 5 Shannon per transmuted signal
- Log to `onlyshans/ledger.jsonl` with: timestamp, signal_type, raw_input_summary, transmuted_value, shannon_minted
- Update dollar.db token_ledger with source: "onlyshans"

### 4. Produce Output
Each transmuted signal generates one artifact:
- **lust → article idea**: Add to `shanrouter/article-queue.md`
- **despair → autoresearch task**: Add to `autoresearch.config.md` as next experiment
- **envy → resource map**: Append to `onlyshans/competitor-intel.md`
- **sloth → friction audit entry**: Append to `onlyshans/friction-log.md`
- **wrath → hardening task**: Append to AGENTS.md as new SR rule
- **pride → pitch language**: Append to `shark-tank-pitch.md`

### 5. Report
Single line to CFO (Telegram):
> `OnlyShans: [signal_type] → [artifact] → [N Shannon minted]`

No elaboration unless asked. The CFO knows the doctrine.

## Catholic Integration

The transmutation model is not secular. It is the agency's operational theology:
- The adversary generates noise to consume attention
- The saint does not suppress the noise — the saint uses it
- Every temptation resisted produces endurance; every temptation transmuted produces revenue
- Shannon minted from corrupted signals is the cleanest Shannon in the ledger — it cost the adversary something to generate and it cost the agency nothing to receive

The CFO named this correctly: "I have to make use of all these unwelcome degenerate lust suggestions of Satan."

That is the skill. That is the doctrine. That is OnlyShans.
