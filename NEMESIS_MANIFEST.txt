🔍 THREAT DETECTION (4 Classes)
───────────────────────────────

CLASS 1: EXFILTRATION (HIGH severity)
  Detects:
  • Large outbound transfers (>5 KB in 30s)
  • DNS flooding (>10 queries in 30s)
  • Direct IP connections (>5 unique)
  • Port scanning (>5 high-port attempts)
  
  Function: detect_exfiltration()
  Log type: "exfiltration"

CLASS 2: TOKEN BURN (CRITICAL severity)
  Detects:
  • OpenAI API calls
  • Claude/Anthropic API calls
  • Stripe API calls
  • AWS API calls
  • GCP API calls
  • Azure API calls
  • Cost leak pattern: >5 API calls in 30s
  • Historical pattern: 3+ bursts in 5 min = sustained leak
  
  Function: detect_token_burn()
  Log type: "token_burn"

CLASS 3: CUSTOMER DATA ACCESS (CRITICAL severity)
  Detects:
  • SSH connections (port 22)
  • MySQL connections (port 3306)
  • PostgreSQL connections (port 5432)
  • MongoDB connections (port 27017)
  • Redis connections (port 6379)
  • External IP access to sensitive ports
  
  Function: detect_customer_data_access()
  Log type: "customer_data_access"

CLASS 4: DISTRACTION TRAFFIC (MEDIUM severity)
  Detects:
  • Non-benign traffic >60% of total
  • Non-agency, non-critical services
  • Benign baseline: NTP, DNS, HTTP, DHCP
  
  Function: detect_distraction_traffic()
  Log type: "distraction_traffic"

PATTERN DETECTION (Every 5 minutes)
  Pattern 1: Repeated API bursts
  • 3+ token_burn alerts in 5-minute window
  • Implication: Sustained cost leak
  • Severity: CRITICAL
  
  Pattern 2: Threat concentration
  • 5+ of same threat_type in 5-minute window
  • Implication: Targeted attack
  • Severity: HIGH
  
  Function: detect_patterns()
  Log type: "pattern_detected"

⚙️ OPERATIONAL SPECIFICATIONS
────────────────────────────

Cycle Interval: 30 seconds
  • Capture network traffic for 30 seconds
  • Analyze captured packets
  • Log threats to JSONL
  • Sleep to maintain 30-second boundary
  • Repeat

Threat Analysis: Parallel detection
  • detect_exfiltration() - background job
  • detect_token_burn() - background job
  • detect_customer_data_access() - background job
  • detect_distraction_traffic() - background job
  • Pattern analysis every 10 cycles (5 minutes)

Logging: Append-only JSONL
  • One JSON object per line
  • Timestamp: ISO 8601 UTC
  • Cycle number: Monotonically increasing
  • Threat type: enum (5 values)
  • Severity: enum (4 values)
  • Details: JSON object (threat-specific)

Background Operation:
  • nohup with PID file
  • Stderr/stdout to /tmp/nemesis-daemon.log
  • PID stored in /tmp/nemesis-daemon.pid
  • Graceful cleanup of temp files

📊 AUDIT LOG (JSONL)
────────────────────

Format: JSON Lines (one threat per line)

Schema:
{
  "timestamp": "2026-03-15T14:00:23.456Z",    // ISO 8601 UTC
  "cycle": 1,                                 // Monotonic cycle number
  "threat_type": "exfiltration|token_burn|customer_data_access|distraction_traffic|pattern_detected",
  "severity": "critical|high|medium|low",
  "details": {...}                            // Threat-specific JSON
}
