## RULE 3: MAKE THY GOLD MULTIPLY

> **Policy:** 60% of revenue reinvested into production. 40% retained as reserves/buffer.

### Strategic Objective
Prove that revenue growth compounds. Money earned = money that accelerates production. Break the "single-project dependency" trap.

### Reinvestment Formula

```
Weekly Revenue ($) 
  × 60% = Reinvestment Pool
    ├─ 40% → Agent Bonuses (morale + incentives)
    ├─ 30% → Infrastructure (BitNet scaling, tools)
    ├─ 20% → Customer Delivery (cloud cost pass-through)
    └─ 10% → Learning/Training
  
  × 40% = Retained (emergency fund, profit)
```

### Example (Week 3, $20K cumulative revenue)

```
W1-3 Total Revenue: $20K
  Reinvestment: $20K × 60% = $12K
    ├─ Bonuses: $12K × 40% = $4.8K (agent morale)
    ├─ Infrastructure: $12K × 30% = $3.6K (BitNet servers, tools)
    ├─ Customer Cloud: $12K × 20% = $2.4K (EC2, pass-through)
    └─ Learning: $12K × 10% = $1.2K (training, books)
  
  Retained: $20K × 40% = $8K (reserves)
```

## RULE 4: GUARD THY TREASURES FROM LOSS

> **Policy:** Daimyo audits every transaction. No unauthorized spend. Account reconciliation daily.

### Strategic Objective
Prevent fraud, waste, and mismanagement. Every dollar in/out is documented. Zero tolerance for mysterious spend.

### Daimyo Audit Checklist (Daily)

```
TRANSACTION AUDIT (Daily, <10 min)
  [ ] All deposits documented (source: customer, refill)
  [ ] All expenses >$5 have invoice/receipt
  [ ] All expenses >$50 have pre-approval
  [ ] No unexpected charges (check for subscription creep)
  [ ] Account balance matches ledger
  [ ] No unauthorized account modifications
```

### Account Reconciliation (Weekly)

```
WEEKLY RECONCILIATION (Every Monday)
  [ ] Bank statement vs. ledger: <2% variance
  [ ] Itemized expenses reviewed (all 100)
  [ ] Missing receipts investigated
  [ ] Fraud alerts checked (0 expected)
  [ ] P&L statement prepared
  [ ] Monthly projection updated
```

### Fraud Prevention Rules

| Risk | Prevention | Detection |
|------|-----------|-----------|
| **Unauthorized spend** | Pre-approval for >$50 | Alert if new vendor |
| **Subscription creep** | Manual renewal only | Alert on recurring charges |
| **Account hijacking** | Monitor IP/location | Alert on unusual access |
| **Embezzlement** | Two-person review (Official + Daimyo) | Monthly audit trail review |
| **Misclassification** | All expenses tied to project/rule | Alert if unclassified |

## RULE 5: MAKE OF THY DWELLING A PROFITABLE INVESTMENT

> **Policy:** Infrastructure pays for itself. Every tool, every system must ROI or be defunded.

### Strategic Objective
Ensure infrastructure is productive, not parasitic. Every $1 spent on infrastructure must generate $2+ in output.

### ROI Measurement Framework

| Tool/System | Initial Cost | Ongoing Cost | ROI Metric | Target | Review |
|-------------|--------------|------------|-----------|--------|--------|
| **BitNet Server** | $0 (local) | $0 | Tokens saved/month | 10K tokens ($300) | W2, W3 |
| **Grok Inference** | $0 (local) | $0 | Throughput (req/s) | 50 req/s | W2, W3 |
| **SQLite Analytics** | $0 (local) | $0 | Query latency | <50ms p95 | W3 |
| **Prometheus Monitoring** | $0 (setup) | $0 | MTTD (alert time) | <5min | W2, ongoing |
| **Bash Automation** | $0 (labor) | $0 | Time saved/week | 10h/week | W2, ongoing |
| **EC2 (Customer Delivery)** | Variable | Pass-through | Revenue per instance | >3x cost | Per project |
| **GitHub Pages** | $0 | $0 | Inbound traffic | 1K visits/month | W2, W3 |

### ROI Calculation

**Example: BitNet Server**
```
Cost: $0/month (local inference, no cloud cost)
Tokens Saved: 10K tokens/month × $0.03 = $300 saved
ROI: 300/0 = ∞ (infinite ROI for local)
Action: Keep + scale
```

**Example: EC2 (Customer Delivery)**
```
Cost: $50/project (estimated)
Revenue: $1K-5K/project
ROI: $1000 / $50 = 20x (target: >3x)
Action: Keep, expand capacity
```

### Defunding Rule

**If any tool/system:**
- Cost >$0 per month AND
- ROI <3x (i.e., cost more than saves) AND
- Has been running >30 days

**Then:** Daimyo files defund request, team has 48h to justify keeping it. If no justification, shut down immediately.

### Success Criteria (Rule 5)

| Metric | Target | Daimyo Proof |
|--------|--------|-------------|
| All tools ROI tracked | 100% | Tool registry |
| ROI >3x | 100% of paid tools | Monthly ROI report |
| Defunded | Any negative ROI | Defund audit trail |
| Local-first | 90% of ops | Cost breakdown |
